# Security Policy

## Supported Versions

Security updates are provided for the latest released version of `br-dev-jobs`.

| Version | Supported |
|---|---|
| Latest | Yes |
| Older releases | No |

## Reporting a Vulnerability

Do not open a public issue with vulnerability details.

Preferred reporting path:

1. Use GitHub private vulnerability reporting if it is enabled for this repository.
2. If private reporting is not available, open a public issue that only asks for a private maintainer contact and does not include exploit details.

Please include:

- Affected version or commit.
- A concise description of the issue.
- Reproduction steps or proof of concept, when safe to share privately.
- Impact and suggested mitigation, if known.

Maintainers will acknowledge valid reports as soon as practical and coordinate a fix before public disclosure.
